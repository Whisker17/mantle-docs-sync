This release is primarily aimed to enable the Mantle Network to support all features of the Ethereum Prague upgrade.


> **Warning**

**🚧 It is a required upgrade for all operators of Mantle mainnet Network. 🚧**


## Mantle v2[​](https://docs-v2.mantle.xyz/nodes/changelogs/v1.0.2#mantle-v2) <a href="#mantle-v2" id="mantle-v2"></a>

This release introduces a new API `optimism_safeHeadAtL1Block` has been added to the op-node to generate succinct ZKP proofs faster.

* Compatible with op-geth v1.3.1 [#219](https://github.com/mantlenetworkio/mantle-v2/pull/219)
* Added `optimism_safeHeadAtL1Block` [#213](https://github.com/mantlenetworkio/mantle-v2/pull/213)

## op-geth[​](https://docs-v2.mantle.xyz/nodes/changelogs/v1.0.2#op-geth) <a href="#op-geth" id="op-geth"></a>

This upgrade adjusts the `eth_maxPriorityFeePerGas` method, which no longer returns a fixed zero value but instead provides a suggested value based on network congestion.

Key EIPs included in this upgrade:

* EIP-1153: Introduces `TSTORE/TLOAD` opcodes to reduce the gas cost of temporary data storage in smart contracts.
* EIP-5656: Introduces the `MCOPY` opcode to optimize memory data copying in smart contracts.
* EIP-6780: Adjusts the `SELFDESTRUCT` opcode
* EIP-7702: Supports `SetCode TX`.
* EIP-2935: Introduces a system contract to store the most recent 8191 historical block hashes.

Changelogs:

* Mantle Prague upgrade [#112](https://github.com/mantlenetworkio/op-geth/pull/112)
