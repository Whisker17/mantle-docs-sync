To build an app on Mantle network, you'll need access to a Mantle node. There are multiple public Node RPC Providers that you can choose from, or you can deploy your own depending on the requirements for your specific use cases.


> **Info**

**This doc is a simple introduction to the process.** For more details, please refer to this:

* [Mantle v2 Skadi mainnet node](https://github.com/mantlenetworkio/networks/blob/main/run-node-mainnetv2)
  

## For Operators Who Deploy Nodes For the First Time <a href="#for-operators-who-deploy-nodes-for-the-first-time" id="for-operators-who-deploy-nodes-for-the-first-time"></a>

### Prerequisites[â€‹](https://docs-v2.mantle.xyz/nodes/verifier#prerequisites) <a href="#prerequisites" id="prerequisites"></a>

Ensure you have the following software installed:

* [Git](https://www.git-scm.com/downloads)
* [Docker](https://docs.docker.com/engine/install/)
* [Node.js](https://nodejs.org/en/download/)
* [Foundry](https://book.getfoundry.sh/getting-started/installation)
* [zstd](https://github.com/facebook/zstd)

### Hardware Recommendations[â€‹](https://docs-v2.mantle.xyz/nodes/verifier#hardware-recommendations) <a href="#hardware-recommendations" id="hardware-recommendations"></a>

For optimal performance, it is recommended to have the following hardware specifications:

* **RAM** - 16GB+
* **CPU** - 8C+
* **Storage** - 4T+ disk(HDD works for now, SSD is better)
* **Bandwidth** - 10mb/s+ download speed

### Approximate Disk Usage[â€‹](https://docs-v2.mantle.xyz/nodes/verifier#approximate-disk-usage) <a href="#approximate-disk-usage" id="approximate-disk-usage"></a>

As of Dec. 25, 2023:

* Archive node: \~1.5TB
* Full node: \~75GB

### Installation and Configuration[â€‹](https://docs-v2.mantle.xyz/nodes/verifier#installation-and-configuration) <a href="#installation-and-configuration" id="installation-and-configuration"></a>

#### Clone the Repository[â€‹](https://docs-v2.mantle.xyz/nodes/verifier#clone-the-repository) <a href="#clone-the-repository" id="clone-the-repository"></a>

Firstly, we need to clone the repository:

```
git clone https://github.com/mantlenetworkio/networks.git
```

#### Initialize to Generate Files[â€‹](https://docs-v2.mantle.xyz/nodes/verifier#initialize-to-generate-files) <a href="#initialize-to-generate-files" id="initialize-to-generate-files"></a>

Before running the node, you need to generate some necessary files, such as the `jwt_secret.txt` and `p2p_node_key.txt`. You can find more details [**here**](https://github.com/mantlenetworkio/networks/blob/main/run-node-mainnetv2#init-to-generate-the-jwt_secret_txt-file-and-the-p2p_node_key_txt).

### Deploy the Node[â€‹](https://docs-v2.mantle.xyz/nodes/verifier#deploy-the-node) <a href="#deploy-the-node" id="deploy-the-node"></a>

#### Download Latest Snapshot From Mantle (Recommended)[â€‹](https://docs-v2.mantle.xyz/nodes/verifier#download-latest-snapshot-from-mantle-recommended) <a href="#download-latest-snapshot-from-mantle-recommended" id="download-latest-snapshot-from-mantle-recommended"></a>

**We recommend that you start the node with the latest snapshot**, so that you don't need to wait a long time to sync data. Check [**this**](https://github.com/mantlenetworkio/networks/blob/main/run-node-mainnetv2#download-latest-snapshot-from-mantle) to know how to get the latest snapshot.


> **Info**


* Depending on the user's location, we provide download links for snapshots in different regions. Check more details [here](https://github.com/mantlenetworkio/networks/blob/main/run-node-mainnetv2#download-latest-snapshot-from-mantle).
* If you want to run with fullnode mode, please change the `--gcmode` to `full` in the [`docker-compose-mainnetv2.yml`](https://github.com/mantlenetworkio/networks/blob/main/docker-compose-mainnetv2.yml#L52).
  

#### Operate Mantle Node[â€‹](https://docs-v2.mantle.xyz/nodes/verifier#operate-mantle-node) <a href="#operate-mantle-node" id="operate-mantle-node"></a>

Once the snapshot is downloaded and extracted to the correct folder, you can start the node, we've prepared a `docker-compose` file for you, so that you can easily start and stop the node.

* [How to start the node](https://github.com/mantlenetworkio/networks/blob/main/run-node-mainnetv2#start)
* [How to stop the node](https://github.com/mantlenetworkio/networks/blob/main/run-node-mainnetv2#stop)
* [How to wipe the node](https://github.com/mantlenetworkio/networks/blob/main/run-node-mainnetv2#wipe)
* [How to view logs](https://github.com/mantlenetworkio/networks/blob/main/run-node-mainnetv2#logs)
* [How to update the node](https://github.com/mantlenetworkio/networks/blob/main/run-node-mainnetv2#update)


> **Info**

Please make sure to change your own L1 RPC Node URL in the [`docker-compose-mainnetv2.yml`](https://github.com/mantlenetworkio/networks/blob/main/docker-compose-mainnetv2.yml).

OP\_NODE\_L1\_ETH\_RPC: 'wss\://mainnet.gateway.tenderly.co' #change this


**Example**[**â€‹**](https://docs-v2.mantle.xyz/nodes/verifier#example)

1. Start the node

```
docker-compose -f docker-compose-mainnetv2.yml up -d
```

2. We can use `docker ps` to get container id and `docker logs` to view the logs

```
docker logs -f <container_id>
```

#### Check Service Status[â€‹](https://docs-v2.mantle.xyz/nodes/verifier#check-service-status) <a href="#check-service-status" id="check-service-status"></a>

Once you start the node, you also need to check the service status, we provide a command to check the service status and the chain data.

* [How to check the service status](https://github.com/mantlenetworkio/networks/blob/main/run-node-mainnetv2#check-service)

**Example**[**â€‹**](https://docs-v2.mantle.xyz/nodes/verifier#example-1)

When we use `cast bn` to check the service status, we will get the block number of the latest block. For example:

```
1758076
```

We can run this command many times to make sure the service is running correctly.
