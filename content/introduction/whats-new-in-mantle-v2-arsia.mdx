---
title: "What's New in Mantle v2 Arsia"
---

The **Arsia Upgrade** is a major release in the evolution of Mantle Network that overhauls the fee model, aligns with the latest OP Stack protocol versions, and introduces DA-aware block management. This upgrade replaces the legacy Bedrock-era fee architecture with a modular, compression-based design and simultaneously activates eight upstream OP Stack forks (Canyon through Jovian).

### Enhancements in Mantle v2 Arsia (v1.5.3) Compared to Limb (v1.4.2)

This section outlines the key improvements and new features in the Arsia Upgrade, drawing from updates in op-geth v1.5.3 and mantle-v2 v1.5.3.

#### Modular Three-Component Fee Model

The most significant change in Arsia is the complete redesign of Mantle's fee structure. The legacy model, which embedded L1 data costs into L2 gas consumption via a `tokenRatio` amplification mechanism, is replaced by a **transparent, modular fee model** where each cost component is independently calculated and charged:

```
TotalFee = L2ExecutionFee + L1DataFee + OperatorFee
```

Key changes:

* **L2 Execution Fee** now reflects **pure EVM execution cost only**. The `tokenRatio` multiplier is removed from gas calculations, and `eth_estimateGas` returns a stable value that no longer fluctuates with L1 fee market conditions.
* **L1 Data Fee** is separated from gas consumption and deducted directly from the sender's balance. It uses a **compression-based estimation** — transaction data is compressed via FastLZ, then a linear regression model approximates the final Brotli-compressed DA size, resulting in more accurate and generally lower L1 data fees.
* **Operator Fee** is a new fee component that provides node operators with a transparent revenue stream, calculated as `gasUsed * operatorFeeScalar * 100 + operatorFeeConstant` and collected by the new `OperatorFeeVault` predeploy.

For the full formula breakdown, see the [Fee Model Handbook (After Arsia)](/system-information/fee-mechanism/fee-model-handbook-after-arsia).

#### Dynamic EIP-1559 Base Fee

Before Arsia, the L2 base fee was effectively static — set by the sequencer without responding to network utilization. Arsia restores a **full dynamic EIP-1559 fee market**:

* The block `extraData` field now encodes EIP-1559 parameters: **denominator**, **elasticity**, and a **minimum base fee**.
* The base fee adjusts dynamically based on block gas usage, providing a real congestion signal.
* A configurable **minimum base fee** floor prevents the base fee from dropping below a governance-set threshold.
* Priority fees are explicitly supported, allowing users to influence transaction ordering during congestion.

#### DA Footprint-Aware Block Management

Arsia introduces a new mechanism to prevent data-heavy blocks from being underpriced, even when execution gas usage is low:

* Each transaction's estimated DA size is converted into a **gas-equivalent DA footprint** using a configurable `daFootprintGasScalar` parameter.
* The block header's `BlobGasUsed` field is repurposed to track per-block DA footprint.
* The DA footprint feeds into the EIP-1559 base fee calculation alongside execution gas, ensuring blocks that are large in data terms are priced accordingly.
* Block building enforces a DA footprint limit, preventing the sequencer from producing blocks that exceed the target DA capacity.

#### OP Stack Fork Alignment

Arsia simultaneously activates the equivalent of **eight upstream Optimism forks** at a single timestamp via `AlignOpWithMantle()`:

| Fork | Key Feature |
|---|---|
| **Canyon** | EIP-1559 denominator adjustment |
| **Delta** | Span batches, Brotli compression for batch data |
| **Ecotone** | Dual-scalar L1 fee model (baseFeeScalar + blobBaseFeeScalar) |
| **Fjord** | FastLZ compression-based L1 data fee estimation |
| **Granite** | Channel timeout refinements |
| **Holocene** | Block-level EIP-1559 parameter encoding in ExtraData |
| **Isthmus** | Operator fee support, updated receipt format |
| **Jovian** | DA footprint gas scalar, full fee model alignment |

The node stack is rebased on **Optimism op-node v1.16.3**, bringing Mantle's protocol layer fully up to date.

#### New RPC API: `eth_estimateTotalFee`

A new RPC endpoint returns the **total estimated transaction cost** including all three fee components:

```
totalFee = L2GasFee + L1DataFee + OperatorFee
```

This replaces the need for developers to manually calculate each component. The corresponding `estimateTotalFee` method is also available in the Mantle-Viem SDK.

<Callout type="info">
Gas estimation behavior has also changed: `eth_estimateGas` no longer applies the pre-Arsia 120% buffer and returns the estimate directly. For accurate total cost, use `eth_estimateTotalFee` instead of manually summing components.
</Callout>

#### Contract Upgrades

At the Arsia activation block, the following L2 contracts are upgraded via deposit transactions:

* **L1Block** — New storage layout with `operatorFeeScalar`, `operatorFeeConstant`, and `daFootprintGasScalar` fields. A new `setL1BlockValuesArsia()` method processes packed calldata with all Arsia-era parameters.
* **GasPriceOracle** — Complete rewrite with compression-based L1 fee calculation (`getL1Fee`, `getL1FeeUpperBound`) and operator fee calculation (`getOperatorFee`). The `isArsia` flag is set at activation.
* **OperatorFeeVault** — New predeploy at `0x420000000000000000000000000000000000001b` for collecting operator fees.

On L1, the **SystemConfig** contract is upgraded with new owner-callable functions:

* `setGasConfigArsia(basefeeScalar, blobbasefeeScalar)` — replaces the deprecated `setGasConfig(overhead, scalar)`
* `setOperatorFeeScalars(scalar, constant)` — configures operator fee parameters
* `setEIP1559Params(denominator, elasticity)` — adjusts dynamic base fee curve
* `setMinBaseFee(minBaseFee)` — sets the base fee floor
* `setDAFootprintGasScalar(scalar)` — controls DA footprint pricing

#### Blob Encoding Migration

Starting with Arsia, the batcher switches from Mantle's custom blob encoding (RLP-encoded frame array spanning multiple blobs) to the **standard OP Stack blob format** (one frame per blob). The derivation pipeline handles this transition seamlessly with dual-mode decoding: it first attempts the legacy Mantle format, and upon failure, falls back to the standard format.

#### L2 Backup Sync

A new `--l2.backup-sync` feature (enabled by default) improves node sync reliability by tracking a backup unsafe L2 head and handling reorgs more gracefully.

### Impact Assessment

**For users**: Transaction fees are now transparently decomposed into three components. Overall pricing should be more predictable, with L2 gas estimates remaining stable regardless of L1 fee market volatility.

**For developers**:
* `eth_estimateGas` now returns **pure L2 execution gas** — do not use it alone for total cost estimation.
* Adopt the new `eth_estimateTotalFee` RPC or the Mantle-Viem SDK's `estimateTotalFee` for accurate total fee estimation.
* The gas estimation no longer includes a 120% buffer.
* See the [Fee Model Handbook](/system-information/fee-mechanism/fee-model-handbook-after-arsia) for detailed formulas and parameter references.

**For node operators**: This is a **mandatory upgrade**. All Mantle Network node operators must update their software before the Arsia activation time. See the [Arsia changelog](/for-node-operators/network-updates/changelogs/mantle-v2-v1.5.3-arsia-upgrade-sepolia) for component versions and upgrade instructions.
